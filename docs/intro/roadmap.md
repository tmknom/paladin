# Paladin 実装ロードマップ

## 1. この文書の役割

この文書は、Paladin の実装をセッション単位の粒度で管理するための順序付き計画です。
大粒度の価値成立段階は `milestones.md` で管理し、本書では日々の実装項目を扱います。

## 2. 全体方針

本ロードマップは、次の方針で進めます。

- 最初は `check` コマンドに集中する
- AST 解析基盤より先に、診断結果の契約を最小限でも固定する
- 最初のルールは 1 つだけ実装し、ルール追加可能な構造を先に作る
- 出力は人間向け表示を持ちつつ、内部的には構造化診断を前提にする
- Ignore、設定ファイル、自動修正、IDE 連携は後段へ回す
- 初期段階では単一ファイル内で判定可能なルールを対象にする

## 3. 運用ルール

- 各項目は 1 セッションで着手・完了判断しやすい粒度にする
- `status.md` で状態を管理する
- 完了後は次の項目へ進む
- スコープが広すぎると判明した項目は分割する

## 4. 実装項目一覧

### R-000 契約モデルを定義する

- **目的**: Paladin が返すべき結果の形を固定し、後続の実装が契約を中心に揃うようにする
- **完了条件**:
    - `status` の 3 状態 (違反なし / 違反あり / 実行失敗) が定義されている
    - 終了コード仕様 (0 / 1 / 2) が固定されている
    - `Violation / CheckSummary / CheckResult` の最小モデルが定義されている
    - 出力仕様の骨格が文書化されている
- **今回やらないこと**: 実際の AST 解析、ルール判定、設定ファイル、Ignore、自動修正
- **関連 milestone**: M1

---

### R-010 Python ファイルを再帰列挙する

- **目的**: 指定されたファイルやディレクトリから、解析対象となる Python ファイル群を安定して取得できるようにする
- **完了条件**:
    - `uv run paladin check src/` のような入力から `.py` 一覧を取得できる
    - 単一ファイルと複数ディレクトリの両方を扱える
    - 同一入力に対して、同じ順序で対象ファイルが得られる
- **今回やらないこと**: AST 生成、ルール判定、Ignore パターン、設定ファイルによる除外
- **関連 milestone**: M2

---

### R-020 AST を生成する

- **目的**: 列挙した各 Python ファイルから、ファイル単位で AST を構築できるようにする
- **完了条件**:
    - 対象ファイルごとに AST が生成できる
    - AST 生成失敗時に `status=error` 相当へ落とし込める
    - AST とファイルパスの対応が保持されている
- **今回やらないこと**: ルールごとの AST 走査、複数ファイルをまたいだ解析、構文エラーの診断出力化
- **関連 milestone**: M2

---

### R-030 ルール実行基盤を作る

- **目的**: AST に対してルール群を適用し、`Violation` を収集する共通基盤を作る
- **完了条件**:
    - `Rule` / `RuleRunner` / `RuleRegistry` の最小インターフェイスが定義されている
    - ルール 1 本が AST を入力にして違反を返せる
    - 複数ファイルへ同じルールを適用できる
    - `RuleRunner` が `Violation` 一覧を収集できる
    - ルール追加時に既存コードの変更範囲が限定される構造になっている
- **今回やらないこと**: 多数のルール追加、ルールごとの設定項目、Ignore コメント、自動修正
- **関連 milestone**: M3

---

### R-040 `check` コマンドの text 出力と終了コードを実装する

- **目的**: `check` コマンドを、実用可能な最小の診断器として end-to-end で成立させる
- **完了条件**:
    - 各違反の対象ファイル、行番号、ルール識別子、違反概要、違反理由、修正の方向性を出力できる
    - 総違反件数、ルール別違反件数、ファイル別違反件数、実行結果の要約 (status) を出力できる
    - 違反が 1 件以上あれば終了コード 1 を返す
    - 違反がなければ終了コード 0 を返す
    - 実行失敗時は終了コード 2 を返す
    - `uv run paladin check src/` が end-to-end で動く
- **今回やらないこと**: JSON 出力、`rules` / `version` コマンド、Ignore、設定ファイル
- **関連 milestone**: M3
