# view モジュール 要件定義書

## 1. 概要

### 目的

`paladin view <RULE_ID>` コマンドで、指定したルールの詳細情報をユーザーに提示する。ルールの意図・見方・修正方向を一度のコマンドで確認できるようにすることで、違反への対処を支援する。

### 解決する課題

- ルール ID だけでは、そのルールが何を意図しているか把握できない
- 違反を修正する際に、ルールの guidance や suggestion をドキュメントで都度探す手間がかかる
- `paladin check` の出力結果からルール詳細へ素早く移動できる手段がない

### システムの概要

`paladin view <RULE_ID>` コマンドを実行すると、指定したルールの詳細情報をラベル付きテキスト形式で出力する。`--format json` オプションを指定すると JSON 形式で出力する。表示される情報は Rule ID・Name・Summary・Intent・Guidance・Suggestion の 6 項目で、テキスト形式では各ラベルは同一の列幅で整列される。

指定した RULE_ID が登録済みルールに存在しない場合は、エラーメッセージを表示して終了する（終了コードは 0）。JSON 形式の場合は `{"error": "..."}` 形式で返す。

## 2. 機能要件

### 2.1 ルール詳細の表示

指定した RULE_ID に対応するルールの詳細情報を表示できる。表示される情報は以下の 6 項目。

- Rule ID（ルール識別子）
- Name（ルール名）
- Summary（概要説明）
- Intent（設計意図）
- Guidance（違反の見方）
- Suggestion（修正方向）

### 2.2 ラベル付き整列出力

詳細情報は各項目をラベル付きで表示し、ラベルと値が揃えられた形式で出力される。ラベル列の幅は最も長いラベルに合わせて固定される。

### 2.3 JSON 形式出力

`--format json` オプションを指定すると、詳細情報を JSON 形式で出力する。JSON オブジェクトには `rule_id`・`rule_name`・`summary`・`intent`・`guidance`・`suggestion` の 6 フィールドを含める。`--format text`（デフォルト）は既存のラベル付きテキスト出力を維持する。

### 2.4 存在しないルール ID への対応

指定した RULE_ID がツールに登録されていない場合は、対象 RULE_ID を含むエラーメッセージを表示する。この場合の終了コードは 0 とする。JSON 形式の場合は `{"error": "..."}` 形式で返す。

## 3. 品質要件

### 3.1 シンプルな操作性

`paladin view <RULE_ID>` というシンプルな形式で詳細を確認できる。`--format` オプションで出力形式を切り替えられる。

### 3.2 出力の一貫性

同一の RULE_ID に対して、常に同一の出力を返す。実行のたびに内容や整列が変動しない。

## 4. 前提条件

### 4.1 技術基盤

- Python 3.11 以上
- paladin パッケージの rule モジュールにルールが登録されていること
- 各ルールが Rule ID・Name・Summary・Intent・Guidance・Suggestion のメタ情報を持つこと
