# list モジュール 要件定義書

## 1. 概要

### 目的

`paladin list` コマンドで、利用可能なルールの一覧をユーザーに提示する。ルールの存在と概要を素早く確認できるようにすることで、静的解析の活用を促進する。

### 解決する課題

- どのようなルールが存在するか、ツールを調べずに把握できない
- ルール ID とルール名の対応を毎回ドキュメントで探す手間がかかる
- 利用可能なルールの全体像を一目で把握する手段がない

### システムの概要

`paladin list` コマンドを実行すると、ツールに登録されているすべてのルールの一覧を出力する。デフォルトはテキスト形式で、`--format json` オプションを指定すると JSON 形式で出力できる。

## 2. 機能要件

### 2.1 ルール一覧の表示

ツールに登録されているすべてのルールを一覧表示できる。表示される情報は以下の通り。

- ルール ID
- ルール名
- 概要説明

### 2.2 整列された出力形式（テキスト）

複数のルールが存在する場合、ルール ID・ルール名・概要説明の各列が揃った形式で出力される。列幅はルール一覧の中で最も長いものに合わせて自動調整される。

### 2.3 JSON 出力形式

`--format json` を指定すると、`{"rules": [{"rule_id": "...", "rule_name": "...", "summary": "..."}, ...]}` 形式の JSON を出力する。登録ルールが存在しない場合は `{"rules": []}` を返す。日本語等のマルチバイト文字はエスケープせずそのまま出力する。

### 2.4 空一覧への対応

登録ルールが存在しない場合、テキスト形式は空の出力（何も表示しない）、JSON 形式は `{"rules": []}` を出力する。

## 3. 品質要件

### 3.1 シンプルな操作性

追加オプションなしで `paladin list` を実行するだけで一覧が確認できる。

### 3.2 出力の一貫性

ルール一覧の出力順序はルールの登録順序に従い、実行のたびに一致する。

## 4. 前提条件

### 4.1 技術基盤

- Python 3.11 以上
- paladin パッケージの rule モジュールにルールが登録されていること
