# list モジュール要件定義

## 概要

### 目的

`paladin list` コマンドで、利用可能なルールの一覧をユーザーに提示する。ルールの存在と概要を素早く確認できるようにすることで、静的解析の活用を促進する。

### 解決する課題

- どのようなルールが存在するか、ツールを調べずに把握できない
- ルール ID とルール名の対応を毎回ドキュメントで探す手間がかかる
- 利用可能なルールの全体像を一目で把握する手段がない

### コンポーネントの概要

`paladin list` コマンドを実行すると、ツールに登録されているすべてのルールの一覧を出力する。`--format` オプションで `text`（デフォルト）と `json` の出力形式を切り替えられる。

## 機能要件

### ルール一覧の表示

各ルールについて以下の情報を表示できる。

- ルール ID
- ルール名
- 概要説明

### 出力形式の切り替え

- `--format` オプションで `text`（デフォルト）と `json` の2種類の出力形式を切り替えられる

### テキスト出力形式

- ルール ID・ルール名・概要説明の3列を整列した形式で出力できる
- 各列の幅はルール一覧の中で最も長いものに合わせて自動調整される
- 登録ルールが存在しない場合は何も出力しない

### JSON 出力形式

- `{"rules": [{"rule_id": "...", "rule_name": "...", "summary": "..."}, ...]}` 形式の JSON を出力できる
- 登録ルールが存在しない場合は `{"rules": []}` を出力できる
- 日本語等のマルチバイト文字はエスケープせずそのまま出力できる

### 終了コード

- 正常終了の場合は終了コード `0` を返す
- エラー（不正な引数、予期しない例外）の場合は終了コード `2` を返す

## 品質要件

### 出力の一貫性

ルール一覧の出力順序はルールの登録順序に従い、実行のたびに一致する。

### 明確なエラー報告

不正な引数や予期しないエラーが発生した場合、即座にエラーとして報告すること。エラーを握りつぶして処理を継続しないこと。

### テスト容易性

ルール一覧の取得ロジックと出力フォーマットの生成ロジックは独立してテストできる。実際のルール登録状態に依存せずに出力形式を検証できる。

## 前提条件

### 技術基盤

- Python 3.11 以上
- paladin パッケージの rule モジュールにルールが登録されていること
