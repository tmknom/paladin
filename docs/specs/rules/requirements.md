# list / view モジュール要件定義

## 概要

### 目的

`list` コマンドおよび `view` コマンドに対応するビジネスロジック層として、利用可能なルールの一覧情報と詳細情報を提供する。CLI からの呼び出しに応じ、登録済みルールのメタ情報を整形した文字列として返す。

### 解決する課題

- 生成 AI や人間が「現在どのルールが有効か」を確認する手段がない
- ルール一覧の取得ロジックとテキスト整形ロジックが分散すると、変更時の影響範囲が広がる

### コンポーネントの概要

- **list モジュール**: 登録済みルールのメタ情報（識別子・名称・概要）を収集し、列幅を揃えたテキスト形式に整形して返す
- **view モジュール**: 指定した rule_id に対応するルールの詳細情報をラベル付きテキスト形式に整形して返す

## 機能要件

### ルール一覧の提供（list コマンド）

- 登録済みルールのメタ情報（識別子・名称・概要）の一覧を取得できる
- 取得したメタ情報を列幅を揃えたテキスト形式に整形して返せる
- ルールが1件も登録されていない場合は空文字列を返せる

### 整形出力の仕様（一覧）

- ルール識別子・ルール名・概要をスペース区切りで1行に並べて出力できる
- 列幅は一覧内の最長文字列に合わせて整列できる

### ルール詳細の提供（view コマンド）

- `view <RULE_ID>` で特定ルールの詳細情報を表示できる
- 詳細情報として Rule ID / Name / Summary / Intent / Guidance / Suggestion をラベル付きで表示できる
- 指定した rule_id が存在しない場合はエラーメッセージを表示できる（exit code 0）

## 品質要件

### 再現性

同一のルール登録状態に対して、常に同一の出力文字列を返すこと。出力内容や列幅の整列が実行ごとに変動しないこと。

### ルールの拡張性

新しいルールを追加しても、list / view モジュール自体の変更が不要であること。ルール登録の変更が出力に自動的に反映されること。

## 前提条件

- `lint` モジュールのルール定義（`Rule` Protocol）を利用してルールのメタ情報を取得する
- 出力形式は現時点ではテキストのみとする
