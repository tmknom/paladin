# max-file-length

## メタ情報

| 項目 | 値 |
|------|-----|
| ルールID | max-file-length |
| 対象 | 単一ファイル |

## 概要

単一ファイルの行数が設定された上限を超えた場合に違反を検出します。プロダクションコードとテストコードで異なる上限を設定できます。

## 背景と意図

ファイルの行数は、コードの健全性を示すシンプルかつ信頼性の高い指標です。100行のファイルと1万行のファイルでは、変更容易性もテスタビリティも根本的に異なります。

行数が増加することには、以下のような問題が伴います。

- **責務の肥大化** — 1つのファイルが多くの責務を持つほど、行数は増える。行数の超過は単一責任原則が破られているサインである
- **変更容易性の低下** — ファイルが大きいほど、変更の影響範囲が広がり、修正コストが増大する
- **テスタビリティの低下** — 行数が多いファイルは依存関係が複雑になりやすく、テストの記述と維持が困難になる
- **認知負荷の増大** — 開発者がファイル全体を把握するために要するコストが、行数に比例して増加する

短いファイルが常に正しいわけではありませんが、長いファイルは間違いなく問題の兆候です。行数に上限を設けることで、設計上の問題を早期に発見する機会を得られます。

テストコードはテストケースが増えるほど行数が増える傾向にあるため、プロダクションコードより緩やかな上限を設定します。

## 診断メッセージ

| フィールド | 内容 |
|-----------|------|
| message | `{path}` は `{length}` 行です。上限は `{limit}` 行です |
| reason | ファイルが長すぎることは、責務の肥大化や設計上の問題を示す兆候です |
| suggestion | ファイルを複数のモジュールに分割することを検討してください |

## 検出パターン

### 違反コード

```python
# src/paladin/check/orchestrator.py（上限300行に対して301行以上の場合）
# ファイルの行数が設定された上限を超えている
class CheckOrchestrator:
    ...
# （以降、行数が上限を超えて続く）
```

### 準拠コード

```python
# src/paladin/check/orchestrator.py（300行以内）
# ファイルの行数が上限内に収まっている
class CheckOrchestrator:
    ...
```

```python
# tests/unit/test_check/test_orchestrator.py（上限500行に対して500行以内）
# テストファイルは max-test-lines の上限が適用される
class TestCheckOrchestrator:
    ...
```

## 検出の補足

### 検出ロジック

1. 対象ファイルがテストファイルかどうかを判定する（「テストファイルの判定」参照）
2. テストファイルであれば `max-test-lines` の値を上限として使用し、それ以外は `max-lines` の値を使用する
3. ファイルの総行数（末尾の改行を含む）が上限を超えた場合に違反を報告する
4. 違反の報告行番号はファイル末尾の行番号とする

### テストファイルの判定

ファイルのパスが `tests/` ディレクトリ配下にある場合、そのファイルをテストファイルとして扱います。具体的には、`pyproject.toml` からの相対パスが `tests/` で始まるファイルが対象です。

### 行数のカウント方法

空行・コメント行・インポート行を含む、ファイルの全行をカウントします。コードの意味的な行数ではなく、テキストとしての行数を基準とすることで、検出ロジックをシンプルに保ちます。

### 報告の粒度

1ファイルにつき最大1件の違反を報告します。違反が検出された場合、ファイル末尾の行で違反として記録します。

### 設定ファイル

このルールは `max-lines` と `max-test-lines` パラメータで行数の上限を指定します。

```toml
[tool.paladin.rule."max-file-length"]
max-lines = 300
max-test-lines = 500
```

| パラメータ | 説明 | デフォルト値 |
|-----------|------|------------|
| `max-lines` | プロダクションコードの行数上限 | `300` |
| `max-test-lines` | テストコードの行数上限 | `500` |

`max-lines` と `max-test-lines` はそれぞれ独立して設定できます。未指定の場合はデフォルト値が使用されます。

ディレクトリ別に上限を変えたい場合は `[[tool.paladin.overrides]]` を使用します。

```toml
[[tool.paladin.overrides]]
files = ["scripts/**"]

[tool.paladin.overrides.rules]
max-file-length = false
```

## 既存ツールとの関係

Ruff は `max-doc-length`（ドキュメント文字列の長さ）と `line-length`（1行の文字数）を設定できますが、ファイル全体の行数を制限する機能は持ちません。Pylint は `max-module-lines`（`C0302`）でモジュール行数の上限を設定できますが、テストコードと非テストコードで異なる設定をするには設定ファイルの工夫が必要です。

Paladin では、プロダクションコードとテストコードを単一の設定で区別できるため、実用的な運用が容易です。また、行数制限はコードの健全性指標として設計ルールとの親和性が高く、Paladin のルールセットに加える価値があります。
